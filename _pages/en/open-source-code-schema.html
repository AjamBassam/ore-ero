Metadata Schema for Open Source Code

The metadata schema for the Open Source Code section.

Data files are hosted on GitHub.

Version 1.0

Mandatory fields are marked with "mandatory".

We've created an example file with only mandatory fields you can use to get started.

  • schemaVersion mandatory [string]

    The version of the ORE data schema for Code used.

  • adminCode mandatory [string]

    Administration code - https://github.com/canada-ca/ore-ero/blob/master/_data/administrations

  • releases mandatory [array]

    Contains objects representing each versioned source code release made available.

    • contact mandatory [array]
      • URL [array]

        French and English URL that can be used to reach the point of contact.

        • en [string]
        • fr [string]
      • email mandatory [string]

        An email address to contact the release.

      • name [string]

        The name of a contact or team for the release.

      • phone [string]

        The phone number to contact a release.

    • date mandatory [array]
      • created mandatory [string]

        The date the release was originally created, in YYYY-MM-DD or ISO 8601 format.

      • lastModified [string]

        The date the release was modified, in YYYY-MM-DD or ISO 8601 format.

      • metadataLastUpdated mandatory [string]

        The date the metadata of the release was last updated, in YYYY-MM-DD or ISO 8601 format.

    • description mandatory [array]

      French and English description of the release.

      • en mandatory [string]
      • fr mandatory [string]
    • downloadURL [array]

      French and English URL where a distribution of the release can be found.

      • en [string]
      • fr [string]
    • homepageURL [array]

      French and English URL of the release homepage.

      • en [string]
      • fr [string]
    • languages [array]

      A list of strings with the names of the programming languages in use on the release.

    • name mandatory [array]

      French and English name of the release.

      • en mandatory [string]
      • fr mandatory [string]
    • organization [array]

      French and English name or organization or component within the department of agency to which the release belong.

      • en [string]
      • fr [string]
    • partners [array]

      An array of objects including an acronym for each agency partnering on the release and the contact email at such agency.

      • URL [array]

        French and English URL to a website that can be used to reach the partner.

        • en [string]
        • fr [string]
      • email [string]

        The email address for the point of contact of the partner.

      • name [array]

        French and English name of the partner.

        • en [string]
        • fr [string]
    • licenses mandatory [array]

      An array of objects containing license details.

      • URL mandatory [array]

        French and English URL of the release license.

        • en mandatory [string]
        • fr mandatory [string]
      • spdxID mandatory [string]

        SPDX License identifier - https://spdx.org/licenses/

    • repositoryURLmandatory [array]

      French and English URL of the public release repository for open source repositories.

      • enmandatory [string]
      • frmandatory [string]
    • relatedCode [array]

      An array of source code, libraries, frameworks, APIs, platforms or other software used in this release.

      • URL [array]

        French and English URL where the software can be found.

        • en [string]
        • fr [string]
      • name [array]

        French and English name of the software used in this release.

        • en [string]
        • fr [string]
    • status [string]

      The development status of the release. For example, Production, Beta, ..

    • tags mandatory [array]

      French and English keywords that will be helpful in discovering and searching for the release.

      • en mandatory [array] A list of English keywords
      • fr mandatory [array] A list of French keywords
    • vcs [string]

      A lowercase string with the name of the version control system that is being used for the release. For example, git, svn, ..

    • version [string]

      The version for this release. For example, 1.0.0
